Installation

Get started with DevTools Pro in minutes. Choose your preferred installation method below.

Prerequisites

  • Node.js 16.0 or higher
  • npm 7.0 or higher (or yarn 1.22+)
  • Git 2.0 or higher

Installation Methods

# Install globally
npm install -g devtools-pro

# Or install locally in your project
npm install --save-dev devtools-pro
# Install globally
yarn global add devtools-pro

# Or install locally in your project
yarn add --dev devtools-pro
# Pull the Docker image
docker pull devtools/pro:latest

# Run with your project mounted
docker run -v $(pwd):/workspace devtools/pro:latest analyze

Verify Installation

After installation, verify that DevTools Pro is working correctly:

devtools --version
# Should output: DevTools Pro v2.1.0

Configuration

Customize DevTools Pro using a `devtools.config.js` file at the root of your project.

Basic Example

module.exports = {
  rules: {
    "no-console": "warn",
    "indent": ["error", 2]
  }
};

Fine-tune DevTools Pro to match your workflow.

devtools.config.js

module.exports = {
  rules: {
    "no-dead-code": true,
    "complexity-threshold": 15,
    "security-audit": true
  },
  exclude: ["node_modules", "dist", "test/"],
  output: "reports/"
}

Custom Rulesets

  • -- Enable/disable rule categories
  • -- Auto-fix supported issues
  • -- Set severity levels (info, warning, error)

Advanced Options

  • ignorePatterns ‚Äì Exclude files/folders from analysis.
  • plugins ‚Äì Add custom analysis plugins.

Custom Rules

Define your own linting or code-quality rules.

Example

module.exports = {
  rules: {
    "no-foo-function": {
      create(context) {
        return {
          Identifier(node) {
            if (node.name === "foo") {
              context.report({ node, message: "Avoid using 'foo'" });
            }
          }
        };
      }
    }
  }
};

Rule Options

  • -- Scope: file / function / project
  • -- Auto-fixable: true / false
  • -- Tags: readability, performance, security

CLI Reference

Complete command-line interface reference for DevTools Pro.

devtools init

Initialize DevTools Pro in your project directory. 

devtools init [options]

Options

--template Use a predefined configuration template (react, vue, node, etc.)
--force Overwrite existing configuration
--interactive Run in interactive mode (default)

Examples

# Initialize with React template
devtools init --template react

# Force overwrite existing config
devtools init --force

# Initialize with custom settings
devtools init --template node --force

devtools analyze

Run code analysis on your project. 

devtools analyze [path] [options]

Options

--rules Specify which rule sets to run (security, performance, style)
--output Output format (json, html, text)
--fix Automatically fix issues where possible
